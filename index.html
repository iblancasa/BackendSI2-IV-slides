¿Qué queríamos hacer?

Un sistema cloud para permitir la comunicación entre une empresa y sus clientes.

Para ello se usa:

  • NodeJS
    • ExpressJS
    • Socket.io
  • MongoDB

¿Qué queríamos hacer?

Tres tipos de usuario:

  • Superusuario
  • Entidad
  • Cliente

Detalles a tener en cuenta:

  • Los clientes solo se pueden comunicar con la empresa
  • El superusuario controla la creación de empresas
  • Varios usuarios pueden ser manejar la misma empresa
  • Más cosas

¿Qué queríamos hacer?

  • Autentificación con Google+
  • Integración con Twitter
  • Posibilidad de subir y compartir ficheros
  • Seguridad (de los mensajes)
  • Estadísticas

¿Qué queríamos hacer?

  • Alguna operación con MongoDB

Aprovisionamiento: Ansible

  • Más sencillo de utilizar que otros
  • Buena integración con servicios cloud
  • Extensa documentación

Otros

PaaS: OpenShift

  • Dispone de acceso SSH
  • Dispone de las herramientas que necesitamos
  • Disponible modalidad gratuita
  • Problemas con otros como Heroku

Otros

IaaS: Azure

  • Flexibilidad, alta escalabilidad...
  • Disponemos de una cuenta
  • Extensa documentación

Otros

Contenderización: Docker

  • Actualización automática del contenedor al hacer push
  • Mucha compatibilidad con distintos SO
  • Posibilidad de utilizar en entornos de producción
  • Aislamiento de recursos
